# Voxie — Agents Plan

## 목적
보컬로이드 서브컬쳐 커뮤니티 카드 아카이브 **Voxie**의 MVP를 빠르게 출시한다.

## 운영 원칙
- 모든 작업은 이 문서를 **단일 소스 오브 트루스(SSOT)** 로 유지한다.
- 작업 시작 전/후 이 문서를 갱신한다.
- MVP 범위를 벗어나는 기능은 별도 섹션에 기록하고 보류한다.
- **TDD 우선**: 테스트 → 구현 → 리팩터 순서를 지킨다.
- **브랜치 전략**: `main` 직접 작업 금지, `dev`에서 작업 후 PR로 병합한다.

---

## MVP 목표 (2주)
**카드 공유 + 덱 생성 + 공유 링크**까지 제공한다.

### 핵심 유저 플로우
1. 유저가 카드 작성
2. 카드 피드에서 탐색/필터
3. 카드 상세 확인
4. 카드 5장을 모아 덱 생성
5. 덱 링크 공유

---

## 개발 절차 (필수)
1. **기능 선택**
2. **유저 행동 목록 제작**
3. **E2E + Unit 테스트 작성**
4. **테스트로 기능 구현 (TDD)**
5. **커밋**
6. **dev 브랜치 푸시**
7. **PR 작성**
8. **CI(TEST) 실행/검증**
9. **main 머지 후 배포**

---

## 코드 품질 규칙
- **Husky**: 커밋 시마다 **타입체킹** 실행

---

## 브랜치 전략
- `main` / `dev` 2-브랜치 전략
- 모든 작업은 `dev`에서 진행 → PR로 `main`에 병합

---

## CI/CD 계획
- `dev` 브랜치에서 작업 후 push
- PR 제출 시 **TEST 실행**으로 검증
- PR 머지(`main`) 시 **배포 진행**

## 배포 전략
- **PM2** 프로세스 매니저 사용
- **Cloudflare Tunnel**로 외부 노출

## 화면 목록 (초안)
- `/` 홈 피드 (카드 리스트)
- `/card/[slug]` 카드 상세
- `/new` 카드 작성
- `/deck/new` 덱 생성
- `/deck/[id]` 덱 상세

---

## 데이터 모델 (초안)
```sql

id uuid primary key

title text not null

-- decks
id uuid primary key
title text not null
card_ids uuid[]
created_at timestamp
```
